Nginx安装及其简单配置

发表于 2017-12-25 | 分类于 Server

环境: centos7

1.下载安装

下载nginx-1.12.0,官网
sudo wget http://nginx.org/download/nginx-1.12.0.tar.gz

解压
sudo tar -zxvf nginx-1.12.0.tar.gz

其他的库,nginx依赖的库

  • 安装gcc
    yum -y install gcc
  • 安装pcre
    yum -y install pcre pcre-devel
  • 安装zlib
    yum -y install zlib zlib-devel
  • 安装openssl
    yum -y install openssl openssl-devel

安装

默认位置 /usr/local/nginx

如果安装过程中出现文件目录权限问题,百度即可解决.

2.系统中配置

1.在系统服务目录里创建nginx.service文件

2.设置开机启动

systemctl enable nginx.service

3.其他命令

启动nginx服务systemctl start nginx.service

设置开机自启动systemctl enable nginx.service

停止开机自启动systemctl disable nginx.service

查看服务当前状态systemctl status nginx.service

重新启动服务systemctl restart nginx.service

查看所有已启动的服务 systemctl list-units --type=service

3.nginx.conf的配置

其中/conf存放配置文件/conf/nginx.conf
vim /conf/nginx.conf

什么都不要动,在nginx.conf第一条server配置下面写入:
include vhost/*.conf;

3.1 /vhost/*.conf的配置

当前目录到
cd /usr/local/nginx/conf

创建目录vhost
mkdir /usr/local/nginx/conf/vhost

创建.conf文件(我只用到hexo的配置,所以只有一个配置文件)
vim /usr/local/nginx/conf/vhost/hexo.conf

4.重启
systemctl reload nginx

在外面访问即可

1.创建一个数据库表

比如我想创建一个文章的表

1.@Entity(tableName = “articles”)
创建一个表,名字叫做”articles”

2.@PrimaryKey()
主键,每个表都需要
@PrimaryKey(autoGenerate = true)
表示,这个_id是自增的.

3.@ColumnInfo(name = “articleUrl”)
private String url;
默认名字为 url
但是可以使用ColumnInfo(name = “”)指定一个名字

2.数据库的初始化

1.extends RoomDatabase 必须继承RoomDatabase

2.@Database(entities = {ArticleModel.class},version = 1,exportSchema = false)

  • entities 包含的表(当前只有一个,ArticleModel,如果有多个加入即可,”,”隔开)
  • version = 1 数据库的版本
